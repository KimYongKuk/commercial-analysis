# Antigravity 사용 가이드: 최고의 코딩 파트너와 협업하기

이 문서는 Google Deepmind 팀이 개발한 AI 코딩 에이전트인 **Antigravity**를 가장 효과적으로 활용하기 위한 매뉴얼입니다. 단순한 챗봇을 넘어, 복잡한 코딩 작업을 수행하고 프로젝트를 주도적으로 관리하는 저와 함께 생산성을 극대화하는 방법을 소개합니다.

## 1. 핵심 기능 및 작동 방식

### 🧠 Agentic Mode (에이전트 모드)
저는 단순한 질의응답을 넘어 **계획(Planning) -> 실행(Execution) -> 검증(Verification)**의 단계로 작업을 수행합니다.
- **Task UI**: 복잡한 작업 시 상단에 현재 작업 상태(Task Status)와 요약(Summary)을 표시하여 진행 상황을 실시간으로 공유합니다.
- **자율성**: 필요한 경우 스스로 터미널 명령어를 실행하거나, 파일을 검색하고, 브라우저를 열어 문서를 확인합니다.

### 📂 Artifacts (산출물 시스템)
작업의 투명성과 체계적인 관리를 위해 다음과 같은 문서를 생성하고 관리합니다.
- **`task.md`**: 작업의 세부 체크리스트를 관리합니다. 진행 상황을 한눈에 볼 수 있습니다.
- **`implementation_plan.md`**: 본격적인 코딩 전, 기술적인 구현 계획을 작성하여 사용자(당신)의 승인을 받습니다.
- **`walkthrough.md`**: 작업 완료 후, 변경 사항과 테스트 결과를 요약하여 보여줍니다.

## 2. 효과적인 협업을 위한 팁

### 🎯 명확한 목표 설정 (Be Specific)
- **Bad**: "이거 고쳐줘."
- **Good**: "`LoginComponent`에서 로그인 버튼을 눌렀을 때 반응이 없는 버그를 수정해줘. 네트워크 탭을 보니 403 에러가 발생하고 있어."
- 구체적인 파일명, 에러 메시지, 기대하는 동작을 알려주시면 더 정확하게 해결할 수 있습니다.

### 📝 계획 단계 활용하기 (Review the Plan)
제가 복잡한 작업을 시작할 때 먼저 **구현 계획(`implementation_plan.md`)**을 제안합니다.
- 이 계획을 꼼꼼히 읽고 피드백을 주세요.
- "이 부분은 라이브러리 A 대신 B를 써줘"라거나 "이 로직은 제외해줘"라고 수정 요청을 하면, 나중에 코드를 다시 짜는 시간을 절약할 수 있습니다.

### 🔄 반복적인 피드백 (Iterative Feedback)
- 작업 중간에 제가 `notify_user`를 통해 확인을 요청할 때가 있습니다.
- 결과물이 마음에 들지 않거나 방향이 틀렸다면 즉시 말씀해 주세요. 저는 피드백을 반영하여 즉시 수정합니다.

### 🎨 웹 개발 & 디자인
- 저는 **Modern Web Design**을 지향합니다.
- "예쁘게 만들어줘"라고 하면 최신 트렌드(Glassmorphism, 다크 모드, 부드러운 애니메이션 등)를 반영하여 디자인합니다.
- 특정 스타일(예: TailwindCSS 사용, 미니멀리즘 등)을 원하시면 미리 말씀해 주세요.

## 3. 주요 명령어 및 요청 예시

| 상황 | 추천 요청 문구 |
| :--- | :--- |
| **새 기능 추가** | "사용자 프로필 페이지를 만들어줘. Next.js를 쓰고, 아바타 업로드 기능이 필요해." |
| **버그 수정** | "지금 빌드가 안 되는데, 터미널 에러 로그를 보고 원인을 찾아서 고쳐줘." |
| **코드 리팩토링** | "`utils.py` 파일이 너무 복잡해. 함수 단위로 분리하고 가독성을 높여줘." |
| **문서화** | "이 프로젝트의 설치 및 실행 방법을 담은 README.md를 작성해줘." |
| **이미지 생성** | "로그인 페이지의 UI 목업 이미지를 그려줘." |

## 4. 주의 사항
- **파일 경로**: 파일을 언급할 때는 가능한 절대 경로를 사용하거나, 파일명을 정확히 명시해 주시면 좋습니다.
- **민감 정보**: API 키나 비밀번호 등 민감한 정보는 코드에 직접 포함되지 않도록 주의해 주세요. (환경 변수 사용 권장)
